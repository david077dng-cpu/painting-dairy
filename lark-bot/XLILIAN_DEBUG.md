# xlilian.art 连接问题诊断报告

## 🔍 问题现象

**症状**：从本地机器无法通过 HTTPS 访问 xlilian.art，但手机可以正常访问

## 🧪 测试结果

| 测试项 | 结果 | 说明 |
|--------|------|------|
| DNS 解析 | ✅ 正常 | xlilian.art → 47.77.237.222 |
| Ping | ✅ 正常 | 延迟约 192ms，无丢包 |
| HTTP (80) | ❌ 连接被拒绝 | 端口未开放或服务未运行 |
| HTTPS (443) | ❌ 连接被拒绝 | 端口未开放或服务未运行 |
| SSH (22) | ✅ 可能正常 | 云服务器通常开放 |

## 🔍 问题分析

根据测试结果，问题出在 **Web 服务器层面**：

1. **服务器本身正常运行** - Ping 通，DNS 正常
2. **但 Web 服务没有运行** - 80/443 端口拒绝连接

### 可能的原因：

#### 1. Web 服务器未运行
- Nginx 或 Apache 没有启动
- 服务异常退出

#### 2. 端口未开放
- 云服务器安全组没有放行 80/443 端口
- 防火墙（ufw/iptables）阻挡了端口

#### 3. 部署问题
- 网站文件不存在或路径错误
- 配置文件错误导致服务启动失败

#### 4. 为什么手机可以访问？
- 可能手机使用了缓存页面
- 或者手机访问时服务器还未出问题
- 也可能是不同的网络路径

## 🛠️ 解决方案

### 方案 1：SSH 登录服务器检查（推荐）

```bash
# 1. SSH 登录服务器
ssh root@47.77.237.222

# 2. 检查 Nginx 状态
systemctl status nginx

# 3. 如果未运行，启动 Nginx
systemctl start nginx
systemctl enable nginx

# 4. 检查错误日志
tail -n 50 /var/log/nginx/error.log

# 5. 检查防火墙
ufw status

# 6. 如果防火墙阻挡了端口，放行
ufw allow 80/tcp
ufw allow 443/tcp
```

### 方案 2：快速诊断命令

在服务器上运行：

```bash
# 检查端口监听
netstat -tlnp | grep -E '80|443|nginx'

# 检查 Nginx 进程
ps aux | grep nginx

# 测试本地访问
curl -I http://localhost

# 检查网站根目录
ls -la /var/www/html  # 或你的实际部署目录
```

### 方案 3：重新部署网站

如果以上方法都无效，可能需要重新部署：

```bash
# 1. 检查部署目录
cd /var/www/html  # 或你的部署目录
ls -la

# 2. 如果有备份，恢复备份
# cp -r backup/* .

# 3. 重启 Nginx
systemctl restart nginx
```

## 📝 检查清单

- [ ] SSH 可以正常登录服务器
- [ ] Nginx 服务正在运行
- [ ] 80 端口已开放（防火墙/安全组）
- [ ] 443 端口已开放（防火墙/安全组）
- [ ] 网站文件存在于正确的目录
- [ ] Nginx 配置文件正确
- [ ] SSL 证书未过期（如果使用 HTTPS）

## 🆘 寻求帮助

如果以上方法都无法解决问题：

1. **检查云服务商状态**
   - 阿里云/腾讯云是否有服务公告
   - 实例是否正常运行

2. **查看监控信息**
   - CPU/内存使用情况
   - 磁盘空间是否已满

3. **联系支持**
   - 提交云服务商工单
   - 查看是否有其他用户遇到类似问题

## 📊 总结

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| HTTPS 连接被拒绝 | Web 服务未运行或端口未开放 | 检查/启动 Nginx，开放防火墙端口 |
| 服务器可以 Ping 通 | 网络层正常，应用层问题 | 排查 Web 服务配置 |
| 手机之前可以访问 | 可能是缓存或之前服务正常 | 恢复 Web 服务运行 |
